# Покос-Обнинск — фронтенд-лендинг для сервиса покоса травы

Одностраничный фронтенд-проект (набор связанных страниц) для компании по покосу травы в Обнинске и Калужской области. Реализованы страницы: главная, услуги и цены, бронирование, отзывы, контакты, акции.

## Стек и структура

- **Стек**: HTML5, CSS3 (Flex/Grid), JavaScript (ES6+), без бэкенда.
- **Библиотеки (CDN)**:
  - Flatpickr — календарь бронирования.
  - Яндекс.Карты API — карта зоны обслуживания и офиса.
  - Font Awesome — иконки.
  - Lightbox2 — просмотр фото «до/после» и фото к отзывам.

Структура:

- `index.html` — главная.
- `services.html` — услуги и цены с калькулятором.
- `booking.html` — онлайн-бронирование с календарём.
- `reviews.html` — отзывы (список и форма добавления, localStorage).
- `contacts.html` — контакты и форма вопросов.
- `promo.html` — акции.
- `css/style.css` — общий современный адаптивный дизайн (mobile first).
- `js/main.js` — бургер-меню, тосты, общие скрипты, последние отзывы на главной.
- `js/calculator.js` — калькулятор цены и передача данных в бронирование.
- `js/booking.js` — календарь flatpickr, форма бронирования, сохранение заявок в localStorage.
- `js/reviews.js` — тестовые отзывы, добавление отзывов, хранение в localStorage.
- `js/map.js` — инициализация карт (зона обслуживания и офис).

## Хранение данных

- **Заявки на бронирование**: сохраняются в `localStorage` под ключом `pokos_bookings`.
- **Отзывы**: первоначальный список и добавленные пользователем отзывы хранятся в `localStorage` под ключом `pokos_reviews`.
- Для демо все формы имитируют отправку — показываются тост-уведомления об успехе/ошибке.

## robots.txt и sitemap.xml

В корне лежат `robots.txt` и `sitemap.xml` для поисковиков. Перед публикацией замените в обоих файлах `https://yoursite.ru` на ваш реальный домен. В `robots.txt` админка и личный кабинет закрыты от индексации.

## Деплой

Проект готов к деплою на любой статический хостинг (Netlify, Vercel, GitHub Pages и т.п.):

1. Скопируйте содержимое папки `project` в корень вашего репозитория/хостинга.
2. Убедитесь, что корневой файл — `index.html`.
3. Относительные пути уже настроены, дополнительных настроек не требуется.
