# Покос-Обнинск — фронтенд, личный кабинет и связь с API

Этот файл описывает, как устроен фронтенд проекта и как он общается с бэкендом (`backend-pokos`) и PostgreSQL.

---

## 1. Быстрый чек‑лист запуска

Каждый раз, когда хочешь потестировать сайт **с личным кабинетом**:

1. Запусти PostgreSQL (обычно служба уже работает).
2. Запусти бэкенд:
   ```powershell
   cd "C:\Users\Vladislaffmaster\backend-pokos"
   npm start
   ```
   В консоли появится: `API запущено на http://localhost:4000`.
3. Запусти статический сервер фронтенда:
   ```powershell
   cd "C:\Users\Vladislaffmaster\project"
   npx serve .
   ```
   В браузере открой `http://localhost:3000/index.html` или сразу `http://localhost:3000/account.html`.
4. Перейди в «Личный кабинет» и пользуйся формами регистрации/входа.

## 3. Личный кабинет (`account.html`)

### Структура страницы

На странице два основных блока:

- `#authSection` — формы:
  - `#loginForm` — вход (email + пароль).
  - `#registerForm` — регистрация (имя, email, телефон, пароль).
- `#accountSection` — личный кабинет (показывается после успешного входа):
  - блок «Ваш профиль» (email, имя, телефон);
  - блок «Клуб лояльности» (уровень, баллы, количество заказов);
  - таблица «Ваши заявки».

В шапке всех страниц есть пункт меню **«Личный кабинет»**, который ведёт на `account.html`.

## 4. Как работает `js/auth.js`

Файл: `project/js/auth.js`

### 4.1. Базовый адрес API

```js
const API_BASE = "http://localhost:4000";
```

Если бэкенд будет перенесён на другой домен или порт, нужно поменять это значение.

### 4.2. Хранение токена

Токен и данные пользователя сохраняются в `localStorage`:

- `pokos_token` — JWT токен.
- `pokos_user` — объект пользователя в JSON.

Функции:

- `getToken()` — достаёт токен из `localStorage`.
- `saveAuth(token, user)` — сохраняет токен и пользователя.
- `clearAuth()` — очищает токен и данные (используется при выходе).

### 4.3. Обёртка для запросов `apiRequest`

```js
async function apiRequest(path, options = {}) {
  const token = getToken();
  const headers = {
    "Content-Type": "application/json",
    ...(options.headers || {}),
  };
  if (token) {
    headers.Authorization = `Bearer ${token}`;
  }

  const res = await fetch(`${API_BASE}${path}`, {
    ...options,
    headers,
  });

  const data = await res.json().catch(() => ({}));
  if (!res.ok) {
    const message = data.error || "Ошибка запроса к серверу";
    throw new Error(message);
  }
  return data;
}
```

Все запросы к API идут через эту функцию. Она автоматически подставляет токен в заголовок `Authorization` (если он есть).

### 4.4. Регистрация

Событие на `#registerForm`:

1. Собирает значения: `email`, `password`, `fullName`, `phone`.
2. Делает запрос:

   ```js
   apiRequest("/api/auth/register", {
     method: "POST",
     body: JSON.stringify({ email, password, fullName, phone }),
   });
   ```

3. В случае успеха:
   - сохраняет токен и пользователя (`saveAuth`);
   - показывает тост «Регистрация успешна…»;
   - прячет блок авторизации `#authSection` и показывает `#accountSection`;
   - вызывает `loadAccountData()`, чтобы подтянуть профиль и заявки.

### 4.5. Вход

Событие на `#loginForm`:

1. Собирает `email`, `password`.
2. Делает запрос:

   ```js
   apiRequest("/api/auth/login", {
     method: "POST",
     body: JSON.stringify({ email, password }),
   });
   ```

3. Дальнейшие шаги такие же, как после регистрации: сохранение токена, показ кабинета и загрузка данных.

### 4.6. Загрузка данных кабинета (`loadAccountData`)

Функция вызывает **два** запроса параллельно:

- `GET /api/me` — профиль + лояльность (таблицы `users` и `loyalty`).
- `GET /api/my-bookings` — список заявок (`bookings`) для текущего пользователя.

Результат:

- Заполняются элементы `#accountEmail`, `#accountName`, `#accountPhone`.
- Заполняется «Клуб лояльности»: `#loyaltyLevel`, `#loyaltyPoints`, `#loyaltyOrders`.
- Таблица `#accountBookingsTable` либо показывает данные по заявкам, либо сообщение `#accountBookingsEmpty`.

### 4.7. Автовход и выход

- При загрузке страницы, если в `localStorage` уже есть `pokos_token`, сразу считается, что пользователь авторизован — блок авторизации скрывается, кабинет показывается и вызывается `loadAccountData()`.
- Кнопка `#logoutBtn` вызывает `clearAuth()` и возвращает пользователя к формам логина/регистрации.

## 5. Маска телефона и валидация email

Маска телефона реализована в `js/main.js`:

- Любой `<input>` с атрибутом `data-phone-mask="true"` автоматически форматируется в вид:
  `+7 (XXX) XXX-XX-XX`.
- Если при расфокусе введено меньше 11 цифр, номер очищается и показывается тост.

Поля email в некоторых формах дополнительно проверяются на наличие `@` и точки в `booking.js` и обработчике формы вопросов в `main.js`.

## 6. Как расширять функциональность

Общий порядок действий, если хочешь добавить новый функционал, завязанный на БД:

1. **Изменить БД** (через `psql`):
   - добавить таблицу или колонку;
   - при необходимости выдать права `pokos_user`.
2. **Добавить/изменить эндпоинт в `backend-pokos/index.js`**:
   - описать новый маршрут `app.get(...)`, `app.post(...)` и т.д.;
   - внутри использовать `pool.query(...)` для работы с PostgreSQL.
3. **На фронте**:
   - в нужном скрипте (`auth.js`, `booking.js`, др.) добавить вызов `apiRequest(...)` на новый эндпоинт;
   - обновить интерфейс (формы, таблицы) под новую логику.

Если придерживаться этой схемы, будет легче ориентироваться и не ломать уже рабочие части.
